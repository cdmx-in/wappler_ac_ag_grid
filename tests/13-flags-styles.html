Suppression flags + ci_sort + hide_filters/sort

Most behavior-suppression flags toggled on at once. hide_filters="email,rating" and hide_sort="department" selectively disable filtering / sorting per column.

rstyles · function condition

Single entry: { condition: 'isRowVip()', customColor: '#fef3c7' }. window.isRowVip(row) returns true when rating ≥ 4.7 and active. Expect ~4 highlighted rows (David, Kate, Olivia, Sam, Xavier).

rstyles · value/operator condition

condition: 'salary > 90000' → light green for high earners, condition: 'salary <= 65000' → light red for low (exercises the 2.1.4 <=/>= parser fix).

rstyles · compound && / ||

active == true && department == Engineering → light blue.

rstyles · Field + plain-value shorthand

{ field: 'department', condition: 'Marketing', customColor: '#fce7f3' } — exact string match on row.department.

rstyles · first-match wins

Two overlapping conditions on the same field. The first entry (salary > 95000 → orange) should win for David (110k) and Kate (105k); the second entry (salary > 90000 → yellow) catches the next band only.

rstyles · legacy object form (backwards compat)

Pre-2.1.4 callers passed an object keyed by field name. The runtime still accepts both shapes (Object.values() normalises to the array path), but rstyles's schema is type: Array, so AppConnect coerces an object-shaped dmx-bind: attribute to [] before the runtime sees it. The legacy code path is only reachable programmatically — this scenario sets rstyles via dmx.app.get('rsLegacy').set(...) and reloads.

cstyles · text + cell area, with >= / <= operators

Two cstyles entries for salary: area:'text' recolors the text (green if ≥ 90k), area:'cell' sets the cell background (red if <= 65k). Exercises the same 2.1.4 operator parsing fix.

data_changes & display_data_changes

data_changes rewrites field values before display. display_data_changes overlays HTML badges on values.

Custom tooltip

custom_tooltip sets an HTML tooltip template; tooltip_show_delay shortens the hover delay.

Live rstyles assertions

Runs in the page after the grids render and counts coloured rows per scenario. The puppeteer runner reads window.__rsCheck and fails the page if any condition didn't apply at least one matching row.
